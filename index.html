Typescript for JavaScript Programmers

Introduction to Typescript

TypeScript stands in an unusual relationship to JavaScript.

For example, JavaScript provides language primitives like string and number, but it doesn’t check that you’ve consistently assigned these. TypeScript does.

This means that your existing working JavaScript code is also TypeScript code. The main benefit of TypeScript is that it can highlight unexpected behavior in your code, lowering the chance of bugs.

This tutorial provides a brief overview of TypeScript, focusing on its type system.

Types by Inference

TypeScript knows the JavaScript language and will generate types for you in many cases. For example in creating a variable and assigning it to a particular value, TypeScript will use the value as its type.

                
                    let helloWorld = "Hello World";

let helloWorld: String

Defining Types

You can use a variety of design patterns in JavaScript.

To cover these cases, TypeScript supports an extension of the JavaScript language, which offers places for you to tell TypeScript what the types should be.

The inferred types for the following variables are name: string and id: number.

                
                    const user = {
                        name: "Sam",
                        id: 0,
                      }

You can explicitly describe this object’s shape using an interface declaration:

                
                    interface User {
                        name: string;
                        id: number;
                      }

You can then declare that a JavaScript object conforms to the shape of your new interface by using syntax like : TypeName after a variable declaration: 

                
                    const user: User = {
                        name: "Hayes",
                        id: 0,
                      }

Typescript will also warn you if you do not follow the conventions of your set interface.

Like JavaScript, Typescript also supports object-oriented programming:

                
                    interface User = {
                        name: string;
                        id: number;
                      }

                      class UserAccount = {
                        name: string;
                        id: number;

                        constructor(name: string, id: number){
                            this.name = name;
                            this.id = id;
                          }

                      }

                      const user: User = new UserAccount("Madison", 1);

Typescript has the following primitive types from JavaScript: boolean, number, bigint, symbol, string, and null. It also adds a few more more such as any(anything), unknown(ensure someone using this type declares what the type is), never(it’s not possible that this type could happen), and void(a function which returns undefined or has no return value).

Composing Types

In Typescript, you can also create your own complex types. You can do this using generics or unions.

Generics: Generics provide variables to types. A common example is an array. An array without generics could contain anything. An array with generics can describe the values that the array contains. 

                
                    type StringArray = Array<string>;
                    type NumberArray = Array<number>;
                    type ObjectWithNameArray = Array<{ name: string }>;

Here is an example of declaring types that use generics:

                
                    interface Backpack <Type> {
                        add: (obj: Type) => void;
                        get: () => Type;
                      }
                    declare const backpack: Backpack<string>;

Unions: With a union, you can declare that a type could be one of many types. For example, you can describe a boolean type as being either true or false: 

                
                    type MyBool = true | false>;

Special tool: to learn the type of a variable, use typeof: Say you have let s = "hello" and you create an if statement such as: if(typeof s === "string") this result should be true.

Structural Type System

One of TypeScript’s core principles is that type checking focuses on the shape that values have. This is sometimes called “duck typing” or “structural typing”.

In a structural type system, if two objects have the same shape, they are considered to be of the same type.

                
                    interface Point {
                        x: number;
                        y: number;
                      }
                    function logPoint(p: Point) {
                        console.log(`${p,x}, ${p,x}`);  
                    }
                    
                    const point = { x: 12, y: 26};
                    logPoint(point);

Looking above, you can see that since point is the same shape as the Point type, it inherits the same properties. The point variable never has to be declared of type Point.

Document Object Model

A Document Object Model(DOM) is a data structure corresponding to the HTML displayed in a web browser. A DOM Tree is used to represent the data structure, made up of nodes. Nodes are individual objects in a tree and represent the elements that make up the document.

Event Handling

An event is an action, usually cause by the user, that the web browser responds to. The code that is executed because of an event is called an event handler or event listener.

Regular Expressions

Programs will often need to recognize patterns in data and a way to recognize a pattern within strings is using regular expressions. A regular expression is a string pattern that is matched against a string.

For example, you can use a regex expression to find if the following string has the set pattern "abc". In code this could be declared with let re = new RegExp("abc"); or let re = /abc/; and your string could be tested using test(str);.

Regex Patterns
Pattern Definition Example
* Match the preceding character 0 or more times. /ab*c/: "abc", "abbbbc", and "ac".
+ Match the preceding character 1 or more times. /ab+c/: "abc" and "abbbbc", NOT "ac".
? Match the preceding character 0 or 1 time. /ab?c/: "abc" and "ac", NOT "abbbbc".
^ Match at the beginning. /^ab/: "abc", NOT "cab".
$ Match at the end. /ab$/: "cab", NOT "abc".
| Match the string on the right or the left. /ab|cd/: "abc" and "bcd"
