Typescript for JavaScript Programmers

Introduction to Typescript

TypeScript stands in an unusual relationship to JavaScript.

For example, JavaScript provides language primitives like string and number, but it doesn’t check that you’ve consistently assigned these. TypeScript does.

This means that your existing working JavaScript code is also TypeScript code. The main benefit of TypeScript is that it can highlight unexpected behavior in your code, lowering the chance of bugs.

This tutorial provides a brief overview of TypeScript, focusing on its type system.

Types by Inference

TypeScript knows the JavaScript language and will generate types for you in many cases. For example in creating a variable and assigning it to a particular value, TypeScript will use the value as its type.

                
                    let helloWorld = "Hello World";

let helloWorld: String

Defining Types

You can use a variety of design patterns in JavaScript.

To cover these cases, TypeScript supports an extension of the JavaScript language, which offers places for you to tell TypeScript what the types should be.

The inferred types for the following variables are name: string and id: number.

                
                    const user = {
                        name: "Sam",
                        id: 0,
                      };

You can explicitly describe this object’s shape using an interface declaration:

                
                    interface User {
                        name: string;
                        id: number;
                      }

You can then declare that a JavaScript object conforms to the shape of your new interface by using syntax like : TypeName after a variable declaration: 

                
                    const user: User = {
                        name: "Hayes",
                        id: 0,
                      };

Composing Types

With TypeScript, you can create complex types by combining simple ones. There are two popular ways to do so: with unions, and with generics.

Document Object Model

A Document Object Model(DOM) is a data structure corresponding to the HTML displayed in a web browser. A DOM Tree is used to represent the data structure, made up of nodes. Nodes are individual objects in a tree and represent the elements that make up the document.

Scope

A JavaScript variable's scope is the context by which a variable can be accessed. The keywords let and var declare variables, but have differing scope. var has function scope and let has block scope.

Another way of determining the scope of a variable is by declaring it in or outside of a function. Declaring a variable in a function would give it local scope, only that function has access to that variable, and any declared outside a function would be global, all functions have access to that variable.

Event Handling

An event is an action, usually cause by the user, that the web browser responds to. The code that is executed because of an event is called an event handler or event listener.
